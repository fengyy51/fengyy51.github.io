一个专注 WEB 开发的技术博客

手把手教你搭博客

  • 2018-08-03

搭建博客有多种方式，近期选择hexo搭建了自己的博客并绑定专属域名，非常激动。此刻记录一下历程。欢迎大家去我的博客做客，有问题的地方请不吝指正。指向我的博客之家：https://fengyy.work

环境准备

安装前提

在建立自己的博客前需要安装安装node.js git，在此就不多加描述了，将地址奉上。

  • node
  • git
  • 设置user.name user.email配置信息：
    • git config –global user.name “你的GitHub用户名”;
    • git config –global user.email “你的GitHub注册邮箱”
  • 生成ssh密钥文件：
    • ssh-keygen -t rsa -C “你的GitHub注册邮箱”，然后直接三个回车即可，默认不需要设置密码。然后找到生成的.ssh的文件夹中的id_rsa.pub密钥，将内容全部复制。打开gitHub_Settings_keys 页面，新建new SSH Key。最后，在Git Bash中检测GitHub公钥设置是否成功，输入 ssh git@github.com。如果操作成功则会显示You’ve successfully authenticated.

安装hexo

  • $ npm install -g hexo-cli

搭建博客

创建repository

  • 在github中创建一个新的repository，将repository name设置为【github昵称】.github.io

本地搭建hexo环境

创建博客目录

在本地创建存放博客的文件夹，在此文件夹下使用如下命令

  • $ hexo init
  • $ npm install

生成静态页面

  • $ hexo clean 用于清除本地public中的内容

  • $ hexo g 生成public中的各文件

运行

  • $ hexo s 启动服务器，成功后即可在浏览器中输入localhost:4000查看效果

更改发布内容

  • 在source/_posts地址下修改markdown文件，此文件即要发布的博客的源地址。

  • 修改后重新生成静态页面、运行，即可看到成型的本地个人博客

更换好看的主题

hexo中有很多主题，可以借助这些主题进行博客的美化。可以去官网查看.

下载主题

  • 本地地址切换到themes目录下，使用git clone 命令需要使用主题的https地址。

修改主题参数

  • 修改_config.yml文件中的theme参数为下载的主题名称。

修改主题默认配置

  • 进入使用的主题目录下，修改其中的_config.yml文件中与用户信息、文件内容相关的配置

将博客发布到线上

修改_config.yml文件

  • 可以修改title description等与博客内容相关的参数

  • 可以修改author等与作者相关的参数

  • url 为可以访问的网址入口，将此参数修改为 https://github昵称.github.io

  • 修改deploy,修改为

deploy type: git
repository: ssh 的repository url
branch: branch名称

部署

  • 安装部署插件，命令为：npm install hexo-deployer-git –save
  • hexo clean
  • hexo g
  • hexo d 将本地博客内容部署到线上

修改域名

购买域名

  • 本人在阿里云购买和名字相关的域名，注意此处不是广告。实名认证后即可使用

绑定域名

  • 进入阿里云的管理控制台的域名列表，找到自己的域名，进入解析。添加两个记录。记录类型均为CNAME，主机记录分别为：www和@。记录值均为github昵称.github.io，解析线路均为默认。

修改本地source目录

  • 在source目录下添加CNAME文件，其中添加购买的域名。亲测过在github中修改setting、手动添加CNAME文件方式均失败，因为在deploy时会将代码修改。最后尝试source文件夹中添加此文件成功，因为source目录中的markdown、html文件会被解析，而其他文件会直接拷贝到public目录下。我们在部署时实际上是将public目录中的内容部署到线上。

访问域名

  • 上述操作完成后，就可以访问你线上美美的博客啦，bingo~
