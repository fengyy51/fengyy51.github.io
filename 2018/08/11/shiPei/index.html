如何进行pc、移动端适配之图片展示

  • 2018-08-11

适配方式

  • 由于在pc端展示的图片采取横版布局，在移动端展示的图片采取竖版布局，所以为了能很好地做到适配，需要从设计的同学那里获取到两种布局的图片。

  • 可以采取两种方式来加载图片。

    • 整体布局依照pc模式即css中图片的url为pc端使用的图片地址，style样式为pc端适用的样式，在js文件中利用useragent来判断是否为移动端，如果是移动端则将图片url替换为移动端使用的图片地址，采取对应的样式。这样的做法会导致在本地测试期间需要刷新页面才可正确显示布局。
    • 在css文件中通过media查询判断设备的宽度和高度尺存来使用不同的图片url，采取不同的样式。

pc端图片响应式布局

  • 采取background的策略，为保证响应式的显示效果设置其background-size为100% auto，background-position为center，background-color设置为背景图片同色，这样就可以保证图片不会在大尺寸屏幕上被拉伸。备注：为其设置背景颜色的dom结构需要保证width100% height100%，否则不会显示背景图片，因为内容为空导致height为0

移动端图片布局

  • 针对需高度还原图片

    • 为了保证图片在各个机型中不出现失真的情况，可以将background-size设置为contain。

  • 针对低质量要求图片

    • 对于只需要在各个机型可以成功展示的图片，可以将background-size设置为100% 100%,这样就可以在各个机型中展示时将图片扩充到整个屏幕中。

  • 由于对图片布局来说，使用background的频率很高。此处附上对这一属性的详细说明：background

移动端实机测试问题

  • 在Chrome开发者工具中的展示效果与实机测试不一定相符，为了解决这种情况，可以使用实机进行本地测试。

  • 移动端测试方法为:

    • 可以借助charles，下载安装后打开proxy proxy settings，port填写端口号（可以使用默认的8888），勾选“enable transparent http proxying”，点击确定。

    • 手机和电脑连接同一个网络，对手机端的无线网络进行设置，进入高级设置后选择手动代理，填写mac端的ip 端口号。之后手机访问需要测试的页面。

    • 如果是需要访问https请求的页面，则设置ssl proxy settings，手机还需下载ssl证书。方法为：在charles中help /ssl proxying/ install charles root。。。点击后弹出对话框，选择确定。手机端输入对话框中的网址即可下载证书。

  • 对于实机测出的问题，可以使用media查询来对此机型进行样式的修改。涉及表单的话，如果出现由于表单中内容过多导致小尺寸机型被遮挡的问题，可以将所有需要填写的内容放在一个div中，设置其固定尺寸，overflow-y设为scroll。
