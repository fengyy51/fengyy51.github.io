git常用命令（逐步增加中。。。）

  • 2018-07-30

本地仓库初始化

  • git init：初始化本地仓库，此时可以使用git命令
  • git clone：将远程代码库以及提交历史添加到本地，此时不需要再次git init操作

配置

  • git config list 显示当前的git配置
  • git config （–global）user.name “xxx” 设置用户名，如果设置global则设为全局
  • git config （–global）user.email “xxx@xx.com“设置用户的邮箱

本地仓库内容相关

  • git add .或者git add -A 可以将当前目录的所有文件保存到暂存区
  • git add (dir) 将指定的文件或者目录保存到暂存区
  • git commit -m “xxx”将暂存区中的内容添加到本地仓库中，并添加描述信息
  • git log 可以查看提交历史，每条记录对应有一个id，可以用来回滚
  • git status 可以查看本地工作目录和暂存区的状态，可以在commit 前后来查看是否还有文件未提交到本地仓库以及文件的修改记录
  • git reset –hard 版本id 可以将工作区的内容会退到指定版本

分支相关

  • git branch 列出本地的所有分支
  • git branch -r 列出远程的所有分支
  • git branch -a 列出本地和远程的所有分支
  • git branch xxx 新建分支，但不切换到新的分支
  • git checkout -b xxx 新建分支，并切换到新的分支
  • git checkout xxx 切换到指定的分支，更新本地的工作区内容
  • git merge xxx 将指定分支合并到当前分支，自动解决冲突
  • git branch -d xxx 删除指定的分支

远程仓库相关

  • git remote 查看所有远程仓库
  • git remote set-url 远程主机名（origin） url 修改远程地址
  • git remote rm origin 删除远程仓库
  • git remote add origin url 添加远程仓库地址
  • git pull 远程主机名 remote-branch ：本地-branch 将指定远程分支的内容拉取并自动解决冲突合并到本地的指定分支，相当于fetch和merge命令的集合体如果不指定本地分支，则默认拉取到当前分支；如果本地分支与远程分支存在追踪关系，则可以省略远程分支名与本地分支名。
  • git push 远程主机名 本地-branch ：remote-branch 将本地分支内容推送到远程分支，不包括标签，如果不指定远程分支，则会推送到与本地分支同名的分支，如果远程分支不存在，则会新建。如果不指定本地分支，则会删除指定的远程分支；如果不指定本地以及远程分支，则会将当前分支推送到远程
  • git push 远程主机名 –delete remote-branch 删除指定的远程分支

标记相关

  • git tag 列出所有的标签
  • git tag -l “xxx” 列出针对索引有关的标签
  • git tag xxx 创建标签
  • git tag -a xxx -m “xxx” 创建携带说明的标签
  • git show xxx 列出标签的详细信息
  • git tag -d xxx 删除指定的标签
  • git push 远程主机名 tagname 将指定的标签推送到远程
  • git push 远程主机名 –tags 将本地新增的标签都推送到远程
